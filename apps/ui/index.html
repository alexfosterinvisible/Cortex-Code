Model Selection

ADW selects models based on model_set in state:

Model Set Commands
Heavy (Opus) /implement, /document, /feature, /bug, /chore, /patch
Base (Sonnet) Everything else (classify, review, commit, etc.)

CLI Commands

Run ADW workflows from the command line

# Install ADW in your project
uv add ../adw-framework

# Verify installation
uv run adw --help
Command Arguments Options Description
adw plan issue_number [adw_id] - Plan implementation for an issue
adw build issue_number adw_id - Build implementation from plan
adw test issue_number adw_id --skip-e2e Run tests with auto-fix
adw review issue_number adw_id --skip-resolution Review implementation against spec
adw document issue_number adw_id - Generate documentation
adw ship issue_number adw_id - Ship changes (merge PR)
adw patch issue_number [adw_id] - Create and implement a patch
adw sdlc issue_number [adw_id] --skip-e2e --skip-resolution Run complete SDLC pipeline
adw zte issue_number [adw_id] --skip-e2e --skip-resolution Zero Touch Execution (SDLC + Ship)
adw monitor - - Run cron monitor for new issues
adw webhook - - Run webhook server

Configuration

Configure ADW for your project

.adw.yaml
# .adw.yaml - ADW Framework Configuration

# REQUIRED: Project identification (usually matches GitHub org/repo)
project_id: "your-org/your-repo"

# PATHS
artifacts_dir: "./artifacts"      # State, logs, worktrees
source_root: "./src"             # Base path for new code

# PORTS - For isolated worktrees (supports 15 concurrent agents)
ports:
  backend_start: 9100
  backend_count: 15
  frontend_start: 9200
  frontend_count: 15

# COMMANDS - Claude command paths (framework + project overrides)
commands:
  - "${ADW_FRAMEWORK}/commands"   # Built-in ADW commands
  - ".claude/commands"            # Project-specific overrides

# APP CONFIG - Project-specific settings (customize for your stack)
app:
  # Directory structure
  backend_dir: "app/server"
  frontend_dir: "app/client"

  # Scripts
  start_script: "scripts/start.sh"
  stop_script: "scripts/stop.sh"
  reset_db_script: "scripts/reset_db.sh"

  # Commands
  test_command: "uv run pytest"
  lint_command: "uv run ruff check"
  build_command: "uv run build"
.env
# Required
ANTHROPIC_API_KEY=sk-ant-xxx

# Optional (uses `gh auth` if not set)
GITHUB_PAT=ghp_xxx

# Optional (defaults to "claude")

ADW ID

8-character unique identifier per workflow instance (e.g., abc12345). Used for:

  • State file location: artifacts/{project_id}/{adw-id}/adw_state.json
  • Worktree path: artifacts/{project_id}/trees/{adw-id}/
  • Deterministic port allocation (hash-based)
